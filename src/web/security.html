SECURITY CONTRACT

What Gravio reads, encrypts, and transmits

This page is the plain-language contract for Gravio CLI behavior. It matches the public manifest at /cli/manifest.json and onboarding verification steps.

Read Locally

  • Directory and file structure metadata for scan signals.
  • Git metadata (tracked files and status signals).
  • Project config metadata (manifest files, lock files, CI and docs signals).

Never Uploaded as Plaintext

  • Raw source code file contents.
  • .env secret values for publish payload construction.

Transmitted to Gravio Cloud

  • Scan signal summary payload to /api/scan-evaluate over HTTPS.
  • Encrypted run envelope to /api/publish over HTTPS.
  • Optional artifact summary (scores and check ids) to /api/scans/artifact over HTTPS.

Encryption Modes

  • Client-side AES-256-GCM envelope encryption before publish.
  • API-key-derived key mode (default).
  • Passphrase-derived key mode with PBKDF2 salt.
  • Raw one-time key mode.

CLI Integrity and Provenance

  • Public manifest endpoint: /cli/manifest.json
  • Includes source repository url, build timestamp, CLI SHA-256, and behavior guarantees.
  • Detached signature fields are reserved in the manifest schema for future publication.
